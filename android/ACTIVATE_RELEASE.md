# How to Activate Your Uploaded AAB (Version 12 - 2.0.2)

## Current Status
Your AAB (version 12, version name 2.0.2) is uploaded but showing as **"Inactive"**. This means it's uploaded but not yet released to any track.

---

## Step-by-Step: Activate Your Release

### Step 1: Go to Production Track

1. In the left sidebar, under **"Test and release"**, click **"Production"**
2. This will take you to the Production release page

### Step 2: Create a New Release

1. You should see a section showing your current production release (if any)
2. Click **"Create new release"** button (usually at the top right or in the main content area)

### Step 3: Select Your Uploaded AAB

1. In the **"App bundles and APKs"** section, you'll see your uploaded AAB (version 12)
2. Click **"Add from library"** or **"Select from uploaded bundles"**
3. Select version **12 (2.0.2)** from the list
4. Click **"Add"** or **"Confirm"**

**Alternative:** If you don't see "Add from library", you may need to:
- Click **"Upload"** again and select the same AAB file
- Or the system may automatically show uploaded but unreleased bundles

### Step 4: Add Release Notes

1. Scroll down to **"Release notes"** section
2. Add notes describing this release, for example:
   ```
   - Updated app signing key
   - Bug fixes and performance improvements
   - Version 2.0.2
   ```
3. You can add notes in multiple languages

### Step 5: Review and Start Rollout

1. Review all information:
   - âœ… AAB version 12 (2.0.2) is selected
   - âœ… Release notes are added
   - âœ… All required information is complete

2. Click **"Save"** to save as draft, or
3. Click **"Review release"** to proceed

4. After review, click **"Start rollout to Production"** (or **"Start rollout"**)

### Step 6: Confirm Rollout

1. You may see a confirmation dialog
2. Confirm the rollout percentage (usually starts at 100% for new releases)
3. Click **"Confirm"** or **"Start rollout"**

---

## What Happens Next?

1. **Status Changes:**
   - The AAB status will change from "Inactive" to "Rolling out" or "Active"
   - It may take a few minutes for the status to update

2. **Review Process:**
   - Google will review your release (usually 1-7 days)
   - You'll receive an email when the review is complete

3. **Live on Play Store:**
   - Once approved, your app will be live with the new version
   - Users will be able to download/update to version 2.0.2

## Troubleshooting

### Issue: "Add from library" not showing

**Solution:**
- The uploaded AAB should appear in the release creation page automatically
- If not, try uploading the AAB again directly in the release creation page
- Make sure you're in the correct track (Production/Testing)

### Issue: Version already in use

**Solution:**
- If version 12 is already used elsewhere, you may need to:
  - Remove it from other tracks first, OR
  - Increment version code and rebuild

### Issue: Can't find the AAB

**Solution:**
- Go back to "All app bundles" page
- Verify version 12 is there
- Click on it to see details
- Then go back to Production and create release
