🔧 Troubleshoot RLS Policies

❌ Current Issue:

permission denied for table users

This suggests the RLS policies are not working correctly or there's an authentication issue.

Step 1: Check Current User Authentication

Step 2: Test Table Access Without RLS

Let's temporarily disable RLS to see if the table itself is accessible:

-- Temporarily disable RLS for testing ALTER TABLE public.user_cvs DISABLE ROW LEVEL SECURITY; -- Test if we can access the table now SELECT * FROM public.user_cvs LIMIT 1;

Step 3: Fix RLS Policies

If the table works without RLS, the issue is with the policies. Let's create simpler, more permissive policies:

-- Re-enable RLS ALTER TABLE public.user_cvs ENABLE ROW LEVEL SECURITY; -- Drop all existing policies DROP POLICY IF EXISTS "Users can view their own CVs" ON public.user_cvs; DROP POLICY IF EXISTS "Users can insert their own CVs" ON public.user_cvs; DROP POLICY IF EXISTS "Users can update their own CVs" ON public.user_cvs; DROP POLICY IF EXISTS "Users can delete their own CVs" ON public.user_cvs; DROP POLICY IF EXISTS "Admins can view all user CVs" ON public.user_cvs; -- Create simpler, more permissive policies CREATE POLICY "Allow all for authenticated users" ON public.user_cvs FOR ALL USING (auth.role() = 'authenticated'); -- Alternative: Allow all operations for now (for testing) -- CREATE POLICY "Allow all operations" ON public.user_cvs -- FOR ALL USING (true); -- Test the policies SELECT * FROM public.user_cvs LIMIT 1;
